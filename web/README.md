# PlugHub Web

Публичный лендинг + защищённая админ-панель для приложения PlugHub.
Лёгкий монолит на **Node.js + Express**, подключается к **той же базе Supabase**,
что и десктоп-клиент.

## Что внутри

```
web/
├── server.js          — Express-сервер (API + отдача статики)
├── public/
│   ├── index.html     — лендинг (Tailwind CDN)
│   ├── admin.html     — админ-панель (вход по паролю)
│   └── admin.js       — логика админки (работает только через /api/*)
├── .env               — секреты (service_role, пароль) — НЕ в git
└── .env.example       — шаблон
```

## Запуск

```bash
cd web
npm install      # один раз
npm start        # http://localhost:3000
```

- Лендинг: <http://localhost:3000/>
- Админка: <http://localhost:3000/admin>

## Как это связано с приложением

- **Пользователи** читаются из таблицы `public.profiles` (те же профили, что
  создаются при входе через Discord в приложении).
- **Премиум-ключи** пишутся в `public.premium_codes` тем же форматом и алфавитом,
  что и серверная функция `generate_premium_codes` (`ABCD-EFGH-JKLM-NPQR`,
  без похожих символов). Поэтому сгенерированный на сайте ключ активируется в
  приложении через `redeem_premium_code` без изменений.
- Доступ к БД идёт через **service_role**-ключ Supabase, который живёт только на
  сервере (`web/.env`) и никогда не попадает в браузер. Админка защищена паролем
  → httpOnly-cookie с HMAC-подписью.

## Автообновление версий

Лендинг сам показывает последнюю версию программы — код трогать не нужно.

1. Соберите новую версию (`npm run package` в корне проекта → `dist/PlugHub-Setup-X.Y.Z.exe`).
2. Залейте `.exe` куда угодно, откуда её можно скачать по прямой ссылке
   (Supabase Storage, Google Drive с прямой ссылкой, любой хостинг/CDN).
3. Откройте `/admin` → секция **«Публикация версии»** → впишите номер версии
   (напр. `1.4.0`), вставьте ссылку(и) на сборку, при желании «что нового» →
   **«Опубликовать версию»**.

Дальше сайт всё делает сам:
- кнопки «Скачать» на лендинге начинают вести на новую сборку;
- появляется плашка «Актуальная версия X.Y.Z» и текст «что нового»;
- открытые вкладки подхватывают новую версию в течение 60 секунд (без перезагрузки).

Версии хранятся в `web/data/releases.json` (создаётся автоматически, в `.gitignore`).
Самая свежая определяется по номеру версии (`1.10.0` новее `1.9.0`). Ненужную версию
можно удалить кнопкой в списке — сайт сразу откатится на предыдущую актуальную.

## Безопасность

- `service_role`-ключ и пароль — только в `web/.env` (в `.gitignore`).
- Пароль сравнивается timing-safe; сессионный токен подписан HMAC-SHA256, TTL 12 ч.
- `SESSION_SECRET` в `.env` желательно заменить на свою длинную случайную строку.
- Для публичного домена поднимайте за HTTPS (тогда добавьте `secure: true` в cookie).
