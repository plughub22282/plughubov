# Daily Drop — дизайн

Дата: 2026-07-10
Статус: спроектировано, ожидает `writing-plans`
Часть более широкой инициативы «Блок 1: план удержания DAU (битмейкеры)» (подпроект 1 из 4: Daily Drop → streak → персонализированная лента → Telegram-пуши).

## Контекст

Сейчас `Home.tsx` показывает три секции: «Для вдохновения» (`featured`, случайная выборка через `pickSample()` на `Math.random()`), «В тренде» (`trending`) и «Топ авторов» (`topAuthors`). Ни одна из них не привязана к календарному дню — при каждом рендере/фильтре набор может быть разным.

Цель — добавить фиксированный блок «Новое сегодня»: одинаковый набор ассетов для всех пользователей в течение суток, жёстко обновляющийся раз в 24 часа, с заметным (но не кричащим) оформлением.

## Отбор контента

- Пул — весь каталог, все виды ассетов (`flp`/`template`/`loop`/`drumkit`/`beat`/`preset`) через уже загруженный `items` из `useLibraryIndex()`. Отдельного запроса к `community_plugins` не требуется — `useLibraryIndex()` уже агрегирует все виды.
- Отбор **не** зависит от `created_at` — это детерминированный случайный сэмпл из всего каталога, а не «реально новый контент». Заголовок «Новое сегодня» читается как «новая подборка на сегодня», а не как «новый в каталоге».
- Размер сэмпла — 5 (верхняя граница из исходного «3–5»).
- Витрина — глобальная: все пользователи видят один и тот же набор в один и тот же календарный день, персонализации нет.

## Детерминированный шаффл

`pickSample()` в `Home.tsx` использует `Math.random()` — не годится, т.к. даёт разные результаты на каждый рендер. Нужна отдельная seeded-функция:

```ts
function mulberry32(seed: number) {
  return () => {
    seed |= 0; seed = (seed + 0x6D2B79F5) | 0
    let t = Math.imul(seed ^ (seed >>> 15), 1 | seed)
    t = (t + Math.imul(t ^ (t >>> 7), 61 | t)) ^ t
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296
  }
}

function hashString(s: string): number {
  let h = 0
  for (let i = 0; i < s.length; i++) h = (Math.imul(31, h) + s.charCodeAt(i)) | 0
  return h
}

function pickDailySample(items: LibraryItem[], count: number, dateKey: string): LibraryItem[] {
  const rand = mulberry32(hashString(dateKey))
  const copy = [...items]
  for (let i = copy.length - 1; i > 0; i--) {
    const j = Math.floor(rand() * (i + 1))
    ;[copy[i], copy[j]] = [copy[j], copy[i]]
  }
  return copy.slice(0, count)
}
```

- `dateKey = new Date().toDateString()` — локальная дата устройства, без сервера/таймзоны.
- Вызывается внутри `useMemo` с деп-массивом `[items]` (дата не в депах намеренно — она меняется только между полными перемонтированиями/навигациями, пересчёт триггерится изменением `items` или сменой вкладки, см. «Обновление в течение дня» ниже).
- Ponytail: локальный `dateKey` без сервера — сознательное упрощение. Побочный эффект: дроп меняется в полночь по времени **устройства пользователя**, а не синхронно у всех в мире. Апгрейд, если понадобится глобально-синхронный дроп, — серверная колонка `daily_drop_date` в UTC.

## Компонент и размещение

Новая секция — первая на странице, над «Для вдохновения» (порядок остальных секций не меняется).

```tsx
<section
  className="rounded-2xl border-2 p-4 -m-4"
  style={{ borderColor: 'rgb(var(--ac) / 0.4)' }}
>
  <div className="flex items-center gap-2">
    <h2 className="text-sm font-semibold text-txt-primary">{t('home.dailyDrop')}</h2>
    <span
      className="rounded-md px-1.5 py-0.5 text-[10px] font-bold uppercase"
      style={{ color: 'rgb(var(--ac))', background: 'rgb(var(--ac) / 0.14)' }}
    >
      {t('home.dailyDropBadge')}
    </span>
  </div>
  <p className="text-[11px] text-txt-muted mt-1 mb-4">{t('home.dailyDropSub')}</p>
  <div className="flex gap-4 overflow-x-auto pb-2">
    {dailyDrop.map((item) => /* та же карточка, что в "Для вдохновения": ItemIcon, w-36, onClick={() => onNavigate(item.tab)} */)}
  </div>
</section>
```

- Заметность — акцентная рамка (`--ac` переменная темы) + текстовый бейдж «СЕГОДНЯ» в заголовке. Без живого таймера обратного отсчёта.
- Карточки и клик-навигация — без изменений от паттерна «Для вдохновения», отдельный превью-плеер не нужен (в «Для вдохновения» его тоже нет).
- Ключи локализации `home.dailyDrop`, `home.dailyDropSub`, `home.dailyDropBadge` — в `i18n.tsx` для ru и en.

## Данные и границы изменений

**Нет изменений** в Supabase-схеме, IPC, `preload.ts`, `main`-процессе и типах `LibraryItem`/`CommunityPlugin`/`Plugin` — вся фича реализуется в `Home.tsx` (плюс `i18n.tsx`) поверх уже загруженных данных из `useLibraryIndex()`.

## Обработка ошибок и граничные случаи

- **`items.length < 5`**: `slice(0, count)` естественно отдаёт меньше элементов, без плейсхолдеров-заглушек.
- **`items.length === 0`**: секция не рендерится (`dailyDrop.length > 0 && <section>...`).
- **`loading`/`!loaded`**: секция не рендерится, пока `items` пуст — отдельного скелетона нет, аналогично остальным секциям Home.tsx.
- **Смена дня при открытом приложении**: пересчёт `dailyDrop` происходит при следующем ре-мемо `items`/навигации, не по таймеру. Ponytail: сознательно без `setInterval`/`visibilitychange`-листенера — редкий кейс (непрерывно открытый Home через полночь), не стоит усложнения. Апгрейд — лёгкий `visibilitychange`-listener, если станет заметной проблемой.

## Вне рамок этого цикла

- Персонализация подборки по пользователю — глобальная витрина по решению из брейнсторминга.
- Отбор по `created_at` / реально новый контент — не в этом цикле (см. «Отбор контента»).
- Серверная синхронизация даты дропа (UTC) — см. Ponytail-примечание выше, апгрейд на будущее.
- Streak-система, персонализированная лента «Для тебя», Telegram-пуши — отдельные последующие циклы (подпроекты 2-4 Блока 1).

## Проверка при реализации

- `npx tsc --noEmit` и `npm run build` — чистый прогон, без новых warning.
- Ручной чек-лист в dev-сборке:
  - Секция «Новое сегодня» отображается первой на Home, с акцентной рамкой и бейджем.
  - Один и тот же набор из 5 карточек на протяжении дня при перезаходах/навигации между вкладками.
  - Клик по карточке дропа ведёт на соответствующую вкладку/категорию, как у карточек «Для вдохновения».
  - Каталог с < 5 элементами — секция показывает сколько есть, без ошибок и пустых слотов.
  - Пустой каталог (`items.length === 0`) — секция не рендерится вовсе.
