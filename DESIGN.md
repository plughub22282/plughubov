## Overview

**North Star: тихая студия ночью.** Тёмная комната с одним осмысленным источником света — холодным графитово-синим акцентом — и всем остальным, что намеренно отступает в тень, чтобы этот свет и то, на что он указывает (статус проверки файла, найденный плагин, состояние поля), несли всё внимание.

PlugHub — реестр (`register: brand`) на всей площади приложения, включая утилитарные экраны (Key Manager, Settings): даже чисто функциональные поверхности держат форму премиального минимала, а не скатываются в «просто интерфейс». Платформа — `web` (Electron), поэтому нет обязательного нативного рулбука — типографика, отступы и компоненты подчиняются только этой системе.

Голос интерфейса — сдержанная уверенность: ни одна поверхность не должна напоминать (а) типовой корпоративный SaaS-дашборд с шаблонной сеткой карточек и дежурными иконками, ни (б) поточный пиратский сайт нулевых с баннерами и мигающими кнопками «скачать». Безопасность (Anti-Spoofing → VirusTotal → карантин) показывается визуально — статусами, а не декларативным текстом.

Весь интерфейс — полный монохром: единственный акцент (Graphite Blue) и тонированные в его же hue нейтрали. Отдельных Secondary/Tertiary-акцентов больше нет — категории в каталоге и маркетплейсе различаются текстом/иконками, а не цветом.

Светлая тема (`[data-theme="pearl"]`) держит ту же ролевую структуру и тот же hue — акцент лишь темнее и чуть более насыщен (`#4C6E8A`/`#355874`), чтобы держать контраст на светлом фоне; это тот же Graphite Blue, а не отдельная палитра.

**Правило использования:** Graphite Blue — это точка, а не заливка. Если акцент занимает больше поверхности, чем текст кнопки/иконки/тонкая обводка/glow, что-то пошло не так.

## Typography

- **Sans** (`-apple-system, BlinkMacSystemFont, Inter, Segoe UI, sans-serif`) — весь интерфейсный текст: заголовки, тело, лейблы, кнопки.
- **Mono** (`JetBrains Mono, Fira Code, Consolas, monospace`) — там, где важна точность/техническая достоверность (версии, хэши, технические идентификаторы), не для декоративного эффекта.
- Масштаб — стандартная шкала Tailwind (`text-xs` … `text-2xl`) плюс один добавленный шаг `2xs` (`0.6875rem` / `1rem` line-height), который заменил разнобой 9/10/11px в бейджах и метаданных одним каноническим шагом.
- Нет отдельной display/headline-шкалы — экраны утилитарны по духу (каталог, менеджер ключей, настройки), поэтому иерархия строится весом (`font-medium` / `font-semibold`) и цветом (`txt-primary` → `txt-secondary` → `txt-muted`), а не ростом кегля.

## Elevation

**Философия — «тихая глубина».** Интерфейс почти плоский по умолчанию: карточка — это тонкая граница + одна мягкая тень (`--shadow-card`), без открыточной многослойности. Явная физичность (`translateY(-3px)`, усиленная тень + glow) появляется только в ответ на намерение пользователя — hover/focus на интерактивных карточках (`card-interactive`), а не как постоянное украшение.

`backdrop-filter: blur()` используется так же избирательно: включается на hover (`.audio-player:hover`, `.volume-control:hover`) — момент, когда пользователь уже смотрит на элемент, — а не постоянно на каждой поверхности.

Акцентный `glow` (`--glow` / `--glow-sm`, мягкая тень цвета Graphite Blue) — не декор, а язык статуса: подсвечивает активную/сфокусированную волну плеера, hover карточки, фокус поля. Если glow виден на элементе, который не активен и не в фокусе — это ошибка использования токена.

## Components

Компонентная философия — **«сдержанная точность»**: минимум декора, акцент используется только там, где нужно направить внимание пользователя (primary-действие, активное состояние, фокус). Скругления сознательно чуть туже дефолтных значений Tailwind (`3xl` = `1.25rem`, а не дефолтные `1.5rem`) — форма собранная, не «мягкая игрушка».

- **`button-primary`** — единственная акцентная кнопка на экране; линейный градиент Graphite Blue + тонкий верхний блик, инсет-тень для объёма. Используется для primary CTA («открыть каталог») и других решающих действий.
- **`button-ghost`** — прозрачный/полупрозрачный фон, граница `neutral-border`; вся вторичная плотность действий, не конкурирует с primary за внимание.
- **`card` / `card-interactive`** — плоская база (граница + одна тень); интерактивная версия добавляет подъём и акцентную рамку/glow только на hover.
- **`input-field` / `select-field` / `search-field`** — единая база оформления полей: тонкая внутренняя тень для «утоплённости», акцентная рамка + мягкое кольцо фокуса (`ring` цвета Graphite Blue при 0.08–0.10 альфы).
- **`badge`** — компактный статус-маркер (использует `status.*` или акцентные роли), не самостоятельный CTA.
- **`tab-button`** (активная/неактивная) — активная вкладка получает заливку `accent/0.12` и цвет текста Graphite Blue; неактивная — нейтральный hover без акцента.
- **`nav-section-header`** — аккордеон-заголовок секции навигации; шеврон поворачивается, тело раскрывается через `grid-template-rows` (без фиксированных px).
- **`track-card`** — облегчённая версия карточки для списков треков/плагинов без backdrop-blur (плотные списки не должны «размываться» построчно).
- **`volume-control` / `audio-player`** — фирменные аудио-компоненты: плеер использует Graphite Blue как единственный активный цвет волны, остальное — приглушённый нейтральный слой.

## Do's and Don'ts

**Do:**
- Использовать Graphite Blue как единственный акцент экрана — точечно, не заливкой.
- Показывать безопасность визуально (статус сканирования, состояние карантина), а не только текстом.
- Держать карточки/поверхности плоскими по умолчанию; физичность (тень, подъём, blur) — только в ответ на hover/focus.
- Уважать `prefers-reduced-motion` — глобально, без исключений (уже реализовано на уровне `*`).
- Использовать `mono` только для технически точного контента (версии, хэши), не для эстетики.

**Don't:**
- Не превращать каталог в шаблонную сетку карточек «SaaS-дашборда» — редкость и курирование важнее объёма плиток.
- Не имитировать пиратский сайт нулевых: без баннерной рекламы, мигающих «скачать»-кнопок, визуального шума.
- Не использовать градиентный текст (`background-clip: text` + градиент) — акцент через вес/цвет, не декоративный градиент.
- Не заводить второй/третий акцентный цвет для категорий или трендов — интерфейс полностью монохромный, различие передаётся текстом/иконкой, а не новым hue.
- Не делать glow/тень постоянным украшением статичного элемента — если ничего не активно и не в фокусе, поверхность плоская.
- Не растягивать скругления обратно к дефолтным Tailwind 1.5rem на `3xl` — собранная форма — часть языка бренда.
